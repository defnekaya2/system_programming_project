# Contributing Guidelines

Thank you for your interest in contributing to this project.

## Branching Strategy
- Do not commit directly to the `main` or `develop` branches.
- Create a feature or fix branch using the format:
  - `feature/<short-description>`
  - `fix/<short-description>`
  - `studentX-weekY-task`

## Commit Messages
- Use clear and meaningful commit messages.
- Follow this format:
  - `feat: short description`
  - `fix: short description`
  - `docs: documentation updates`
  - `chore: project maintenance`

## Pull Requests
- Every change must be submitted via a Pull Request.
- Pull Requests must be reviewed by at least one team member before merging.
- CI checks must pass before merging.

## Code Style
- Keep the code clean and readable.
- Remove unused files and unnecessary code.
- Add comments where necessary.

## Testing
- Ensure the project builds successfully before submitting a Pull Request.
- Do not merge broken or failing code.

## Documentation
- Update documentation when making changes that affect usage or setup.
